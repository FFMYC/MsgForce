\## README.md



```markdown

\# MsgForce - 强制公告推送系统



一个轻量级的公告推送系统，管理员可强制推送公告到客户端，客户端收到公告后自动全屏显示，用户必须提交反馈或设置稍后提醒才能退出。



\## 功能特点



\- 🔐 \*\*强制推送\*\* - 公告实时推送到所有在线客户端

\- 📢 \*\*自动全屏\*\* - 收到公告自动全屏显示，无法跳过

\- 💬 \*\*反馈机制\*\* - 用户必须提交反馈才能退出全屏

\- ⏰ \*\*稍后提醒\*\* - 支持设置提醒时间，到时自动重新弹出

\- 🔌 \*\*多服务器\*\* - 支持内网/公网自动切换

\- 🚀 \*\*开机自启\*\* - 客户端支持开机自启动

\- 🛡️ \*\*防关闭\*\* - 用户无法手动关闭程序

\## 安装部署



\### 1. 服务器端部署



\*\*安装 Node.js 依赖：\*\*

```bash

npm install express ws

```



\*\*修改 IP 和端口（编辑 config.json）：\*\*

```json

{

&#x20;   "ip": "0.0.0.0",

&#x20;   "port": 12345

}

```



\*\*启动服务器：\*\*

```bash

node server.js

```



\### 2. 客户端部署



\*\*安装 Python 依赖：\*\*

```bash

pip install websocket-client requests

```



\*\*打包成 exe（可选）：\*\*

```bash

pyinstaller --onefile --noconsole user.py

```



\## 配置说明



\### 修改服务器 IP 和端口



\*\*方法一：修改 config.json\*\*

```json

{

&#x20;   "ip": "0.0.0.0",      # 监听所有网卡

&#x20;   "port": 40049         # 修改端口号

}

```



\*\*方法二：命令行启动时指定端口\*\*

```bash

PORT=40049 node server.js

```



\### 客户端修改服务器地址



编辑 `user.py` 中的 `SERVER\_LIST`：



```python

SERVER\_LIST = \[

&#x20;   "192.168.1.1:12345",      # 内网地址

&#x20;   "127.0.0.1:12345",        # 本机地址

&#x20;   "your-domain.com:40049"   # 公网地址

]

```



\## 使用方法



\### 管理员端



1\. 启动服务器：`node server.js`

2\. 浏览器访问：`http://服务器IP:端口`

3\. 输入公告内容，点击"发布公告"

4\. 查看用户反馈



\### 用户端



1\. 运行客户端程序（双击 exe 或 `python user.py`）

2\. 程序自动最小化到任务栏

3\. 收到公告自动全屏弹出

4\. 输入反馈或设置稍后提醒



\### 紧急退出



连续按 \*\*5 次 ESC\*\* 键可紧急退出程序（仅限维护使用）

\## 常见问题



\### Q: 客户端无法连接服务器？

A: 检查防火墙是否开放端口，确认 IP 和端口配置正确。



\### Q: 如何让客户端开机自启？

A: 打包成 exe 后首次运行会自动添加到注册表。



\### Q: 用户关闭了程序怎么办？

A: 程序没有关闭按钮，只能通过任务管理器或 ESC 紧急退出。



\### Q: 支持多用户同时连接吗？

A: 支持，WebSocket 会广播给所有在线客户端。
